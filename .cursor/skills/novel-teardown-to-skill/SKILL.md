# 网文拆书 v2 → 写作 Skill 蒸馏

## 合规前提（必守）

- **只处理你有权使用的文本**：正版购买、平台授权章节、作者公开片段、或你原创的仿写练习。禁止以本技能为由抓取盗版站全文、传播镜像链接或粘贴大段版权作品。
- 输入以「节选 + 章节定位」为宜（例如自行摘录的前 3 章关键段，每段数百至一两千字），**长尾复制粘贴**易侵权，应缩短为由你概述的情节骨相。

## 三种拆书模式

### 模式一：拆开头（`POST /api/teardown/opening`）

分析作品开头的写作技巧，聚焦前1～3章。

**输入**：
- `excerpt`：正文节选（≥80字，建议前1～3章）
- `book_title`：书名（可选）
- `author`：作者（可选）
- `tags`：用户标签列表（用于匹配主题系统）
- `temperature`：温度（默认0.35）

**输出维度**：
1. **开篇钩子设计**：前300字的手法、翻页动机、裂口呈现
2. **入戏节奏与结构**：节奏曲线、爽点位置、章末钩子类型、认知递进层数
3. **笔法与叙事技巧**：句式比例、对话描写比、叙事距离、信息露出方式
4. **人物立人设技巧**：主角如何立住、配角出场方式、人物关系张力
5. **可执行规则（Do/Don't）**：3～5条 Do + 3～5条 Don't，可独立执行
6. **开头模板（填空式）**：可复用的开头模板

**标签 → 主题匹配**：
- 传入的 `tags` 自动与 `themes.json` 中的 35+ 个主题匹配
- 匹配结果包含 `matched_themes`（已有主题）和可新建的主题
- 同义词扩展：如「都市」自动匹配「职场」「现代」「城市」等

### 模式二：蒸馏作者（`POST /api/teardown/distill-author`）

从作品全文/大段节选中蒸馏作者风格画像，生成独立 SKILL。

**输入**：
- `excerpt`：正文内容（≥200字，越长越准确；支持 txt 文件上传）
- `book_title`：书名（可选）
- `author_name`：作者署名（必填）
- `tags`：用户标签列表
- `temperature`：温度（默认0.38）

**输出**：
1. **作者经历推测**：职业、地域、知识领域、情感模式
2. **笔法特征**：句式偏好、叙事视角、信息密度、环境描写、对话风格、比喻修辞、幽默类型
3. **用词特征**：高频用词（10～20个）、禁用词倾向、语气词习惯、文风标签（3～5个）
4. **节奏与结构习惯**：章节长度偏好、节奏控制手法、章末收束习惯
5. **情感滤光镜**：作者内在驱动力、偏好情感类型、情感处理方式
6. **虚拟作者人设卡片**：文风标签 + 叙事滤光 + 笔法摘要 + 情感基调

**SKILL 生成**：
- 蒸馏完成后自动生成独立的 `SKILL.md` 内容（`skill_content` 字段）
- 保存到 `UserData/kb/*.md`，写作台勾选即可调用
- SKILL 包含：画像速览、虚拟作者人设卡片、详细蒸馏报告、写作时注入规则

**写入记忆宫殿**：
- 蒸馏结果可一键写入记忆宫殿「风格」房间
- 供后续创作参考，虚拟作者人设与写作风格自动注入

### 模式三：经典长篇拆书（`POST /api/teardown/novel`）

沿用 oh-story-claudecode 的 story-long-analyze 框架。

**输入**：
- `excerpt`：正文节选（≥80字）
- `book_title`：书名（可选）
- `mode`：`quick`（快速）或 `deep`（深度）
- `excerpt_note`：节选说明（可选，如"第1～3章"）
- `temperature`：温度（默认0.35）

**输出**：按 story-long-analyze 框架的 Phase 2～4 模板输出。

## 与主题系统联动

### 标签匹配（`POST /api/teardown/match-tags`）

- 传入标签列表，自动与 `themes.json` 中 35+ 个主题匹配
- 同义词扩展：如「玄幻」→「奇幻」「魔幻」；「言情」→「情感」「恋爱」
- 返回 `matched_themes`（已有主题）和 `unmatched_tags`（需新建主题）
- 若有未匹配标签，系统自动生成新主题配置（`teardown_` 前缀）

### 自动新建主题

当用户标签无法匹配任何已有主题时：
- 自动生成 `teardown_<slug>` 格式的新主题
- label 格式：`拆书·<标签1>, <标签2>, ...`
- description 包含来源作品信息

## 与 AI-writer 仓库协作时

### API 端点

| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/teardown/opening` | POST | 拆开头：分析写作技巧 |
| `/api/teardown/distill-author` | POST | 蒸馏作者：生成 SKILL |
| `/api/teardown/novel` | POST | 经典长篇拆书 |
| `/api/teardown/match-tags` | POST | 标签匹配主题 |
| `/api/teardown/write-memory` | POST | 写入记忆宫殿 |
| `/api/teardown/save-skill` | POST | 保存 SKILL 到 kb |

### 前端 UI

- **拆书 tab** 下分三个子页签：拆开头 / 蒸馏作者 / 长篇拆书（经典）
- 拆开头和蒸馏作者均支持：
  - 标签输入（逗号分隔），实时显示主题匹配结果
  - 生成结果一键保存到知识库
  - 生成结果一键写入记忆宫殿
- 蒸馏作者额外支持：
  - 上传 `.txt` 文件（限 2MB）
  - SKILL 文件名自动生成

### 写入记忆

- 拆开头结果写入记忆宫殿「风格」房间
- 蒸馏作者结果写入记忆宫殿「风格」房间（含虚拟作者人设卡片）
- 记忆内容可在写作台「长期记忆」区域查看和管理
- 若指定 `book_id`，记忆写入该书本的独立记忆宫殿

### 与 writer.md / pipeline 合同

- 蒸馏作者生成的 SKILL 可作为 kb 文件参与全书生成与续写
- 虚拟作者人设卡片与 `author_persona.py` 的「虚拟作者」功能互补：
  - `author_persona.py`：随机生成虚拟作者（开笔时自动创建）
  - 蒸馏作者：基于真实作品分析，风格更精确
- 若同时存在两种虚拟作者，以用户全书项目说明为最高优先级
- 深度改 writer/pipeline 合同：优先叠用 `ai-writer-prompt-iterate` 技能

### 合并蒸馏（同一作者多篇作品）

同一作者可能有多篇作品，每篇作品蒸馏后自动保存到 `UserData/author_distills/`。

**流程**：
1. 蒸馏第一篇作品 → 自动保存记录（含详细报告 md 文件 + 索引）
2. 蒸馏同一作者的第二篇作品 → 自动保存，前端「已蒸馏作品」列表刷新
3. 在前端勾选 ≥2 篇作品 → 点击「合并蒸馏」→ 调用 LLM 将多篇蒸馏合并为综合画像
4. 合并后的综合画像可保存 SKILL 或写入记忆宫殿

**合并规则**（由 LLM 执行）：
- 去重：相同特征只保留一次
- 取交集：多篇反复出现的特征权重最高（稳定风格）
- 取并集：不同作品展现的不同面向都要纳入
- 标注矛盾：若不同作品的蒸馏明显矛盾，会在报告中标注

**新增 API**：

| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/teardown/distill-authors` | GET | 列出所有已蒸馏的作者及作品数 |
| `/api/teardown/distill-history` | GET | 列出某作者的所有蒸馏记录 |
| `/api/teardown/distill-detail` | GET | 读取某次蒸馏的详细报告 |
| `/api/teardown/merge-distill` | POST | 合并同一作者的多篇蒸馏 |

**存储结构**：
```
UserData/
  author_distills/
    _index.json          # 索引：{author_name: [records...]}
    <author-slug>/       # 按作者分目录
      <record-id>.md     # 详细蒸馏报告
```

## 自检

- [ ] 输入来源合法，无盗版全文。
- [ ] 输出以「规则 + 结构」为主，少复述原作情节。
- [ ] Do/Don't 可被独立执行，不依赖「读过原书才懂」的梗。
- [ ] 标签匹配覆盖主要中文网文类型。
- [ ] 蒸馏作者 SKILL 可独立在写作台调用。
- [ ] 记忆写入不与现有记忆宫殿结构冲突。
